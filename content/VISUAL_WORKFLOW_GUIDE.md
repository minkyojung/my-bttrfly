# Obsidian 블로그 워크플로우 시각화 가이드

이 문서는 Obsidian 블로그 워크플로우를 시각적으로 이해하기 쉽게 설명합니다.

---

## 전체 워크플로우 개요

```
아이디어 💡
    ↓
빠른 캡처 (⌘⇧I)
    ↓
ideas.md 저장
    ↓
발전시킬 준비가 되면
    ↓
새 포스트 생성 (⌘N)
    ↓
템플릿 자동 적용
    ↓
작성 (Draft)
    ↓
이미지 추가 (⌘V)
    ↓
퇴고 & Linter
    ↓
발행 체크리스트
    ↓
status: published
    ↓
Git Commit (⌘⇧C)
    ↓
Git Push (⌘⇧P)
    ↓
블로그 배포 ✅
```

---

## 1. 아이디어 캡처 워크플로우

### Before: 전통적인 방법
```
아이디어 떠오름
    ↓
메모앱 열기
    ↓
메모 작성
    ↓
나중에 찾기 어려움 😰
    ↓
결국 잊어버림 ❌
```

### After: Obsidian 워크플로우
```
아이디어 떠오름
    ↓
⌘⇧I (QuickAdd)
    ↓
제목 입력 → 내용 입력
    ↓
ideas.md에 자동 저장
    ↓
Dashboard에서 언제든 확인 ✅
    ↓
Git 자동 백업 🔒
```

**시간 절약**: 5분 → 30초

---

## 2. 블로그 포스트 작성 워크플로우

### 단계별 프로세스

```
┌─────────────────────────────────────────────┐
│ 1️⃣ 새 포스트 시작                            │
│    ⌘N (QuickAdd: 새 블로그 포스트)            │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 2️⃣ 정보 입력                                 │
│    • 한글 제목: "초보자가 되는 불편함"         │
│    • 영문 슬러그: "beginner"                  │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 3️⃣ 자동 생성                                 │
│    • 파일: 2024-03-24-beginner.md            │
│    • Frontmatter: title, date, tags, status  │
│    • 본문 구조: 템플릿 자동 적용              │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 4️⃣ 작성                                      │
│    • 본문 작성                                │
│    • 이미지 복사-붙여넣기 (⌘V)                │
│    • 내부 링크 [[2024-03-15-other-post]]     │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 5️⃣ 퇴고                                      │
│    • 맞춤법 검사                              │
│    • Linter 자동 실행 (저장 시)               │
│    • 미리보기 확인 (⌘E)                       │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 6️⃣ 발행 준비                                 │
│    • publish-checklist.md 확인               │
│    • status: draft → published               │
└──────────────┬──────────────────────────────┘
               ↓
┌─────────────────────────────────────────────┐
│ 7️⃣ 배포                                      │
│    • ⌘⇧C (Git Commit)                        │
│    • ⌘⇧P (Git Push)                          │
│    • 자동 빌드 & 배포                         │
└─────────────────────────────────────────────┘
```

## 3. 이미지 처리 워크플로우

### Before: 수동 처리
```
이미지 준비
    ↓
Finder에서 public/images/ 찾기
    ↓
파일 복사
    ↓
이름 수동 변경
    ↓
마크다운 경로 수동 입력
    ↓
오타 가능성 😰
```

### After: 자동 처리
```
이미지 복사 (⌘C)
    ↓
에디터에서 ⌘V
    ↓
┌───────────────────────────────┐
│ Paste Image Rename 자동 처리   │
│ • 파일명: beginner-20240324... │
│ • 저장: /public/images/posts/  │
│ • 경로: 자동 생성               │
└───────────────────────────────┘
    ↓
완료! ✅
```

**시간 절약**: 2분 → 5초

### Dashboard의 역할

```
┌─────────────────────────────────────────────┐
│          📊 Dashboard.md                     │
├─────────────────────────────────────────────┤
│                                              │
│  ┌────────────────────────────────────┐     │
│  │ 📈 통계                             │     │
│  │ • 총 포스트: 24개                   │     │
│  │ • 발행: 20개                        │     │
│  │ • 초안: 4개                         │     │
│  │ • 이번 달: 3개                      │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │ 📝 초안 목록                        │     │
│  │ 1. [새로운 아이디어] - 50% 완성    │     │
│  │ 2. [기술 블로그] - 20% 완성         │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │ 🏷️ 태그 인기도                      │     │
│  │ • 일 (8개)                          │     │
│  │ • 사고 (6개)                        │     │
│  │ • 기술 (5개)                        │     │
│  └────────────────────────────────────┘     │
│                                              │
│  ┌────────────────────────────────────┐     │
│  │ 🎯 이번 주 목표                     │     │
│  │ ✅ 주 1회 발행 달성!                │     │
│  └────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```

### 사용 시나리오

**시나리오 1: 월요일 아침**
```
Dashboard.md 열기
    ↓
이번 주 목표 확인
    ↓
초안 목록에서 작업할 포스트 선택
    ↓
작성 시작
```

**시나리오 2: 주말 회고**
```
Dashboard.md 열기
    ↓
월별 포스팅 현황 확인
    ↓
태그 인기도 분석
    ↓
다음 달 계획 수립
```

## 5. Git 자동 백업 시스템

### 자동 백업 타임라인

```
09:00 - 블로그 작성 시작
    ↓
09:15 - 아이디어 캡처 (⌘⇧I)
    ↓
09:30 - 🔄 자동 커밋 & 푸시 (첫 번째)
    ↓
10:00 - 새 포스트 작성 (⌘N)
    ↓
10:30 - 🔄 자동 커밋 & 푸시 (두 번째)
    ↓
11:00 - 이미지 추가
    ↓
11:30 - 🔄 자동 커밋 & 푸시 (세 번째)
    ↓
12:00 - 발행 준비
    ↓
12:01 - ⌘⇧C + ⌘⇧P (수동 커밋)
    ↓
✅ 모든 변경사항 안전하게 백업됨
```

### 안전망

```
┌─────────────────────────────────────────────┐
│        자동 백업 (30분마다)                  │
│              ↓                               │
│        ┌────────────┐                        │
│        │  Obsidian  │                        │
│        └─────┬──────┘                        │
│              ↓                               │
│        Obsidian Git                          │
│              ↓                               │
│        ┌────────────┐                        │
│        │   GitHub   │ ← 클라우드 백업        │
│        └────────────┘                        │
│                                              │
│  만약 파일 손실되면:                         │
│  1. GitHub에서 복구                          │
│  2. Git 히스토리로 이전 버전 복원            │
│  3. 30분 이내 변경사항만 손실                │
└─────────────────────────────────────────────┘
```

## 6. 태그 시스템 구조

### 태그 계층 구조

```
┌─────────────────────────────────────────────┐
│              태그 시스템                     │
├─────────────────────────────────────────────┤
│                                              │
│  Level 1: 카테고리 (1개만 선택)             │
│  ┌──────────────────────────────────┐       │
│  │  일   사고   기술   삶            │       │
│  └──────────────────────────────────┘       │
│             ↓                                │
│  Level 2: 서브 카테고리 (여러 개)           │
│  ┌──────────────────────────────────┐       │
│  │  스타트업, 의사결정, 성장        │       │
│  │  Python, React, DevOps           │       │
│  │  독서, 여행, 취미                │       │
│  └──────────────────────────────────┘       │
│                                              │
└─────────────────────────────────────────────┘
```

### 예시

```yaml
# 포스트 1
tags:
  - 일              # 메인 카테고리
  - 스타트업        # 서브 카테고리
  - 의사결정

# 포스트 2
tags:
  - 기술            # 메인 카테고리
  - Python
  - DevOps
```

### Dashboard에서 자동 분류

```
🏷️ 일 (8개)
  ├─ 스타트업 (5개)
  ├─ 의사결정 (3개)
  └─ 성장 (2개)

🏷️ 기술 (6개)
  ├─ Python (3개)
  ├─ React (2개)
  └─ DevOps (1개)
```

## 7. 단축키 워크플로우 맵

### 일상적인 작업

```
┌─────────────────────────────────────────────┐
│        자주 사용하는 단축키                  │
├─────────────────────────────────────────────┤
│                                              │
│  📝 컨텐츠 생성                              │
│  ⌘N         새 블로그 포스트                 │
│  ⌘⇧I        아이디어 캡처                    │
│  ⌘V         이미지 붙여넣기                  │
│                                              │
│  🔍 탐색                                     │
│  ⌘O         파일 빠른 열기                   │
│  ⌘P         명령 팔레트                      │
│  ⌘⇧F        전체 검색                        │
│                                              │
│  ✏️ 편집                                      │
│  ⌘E         편집/미리보기 토글               │
│  ⌘B         굵게                             │
│  ⌘K         링크 삽입                        │
│                                              │
│  💾 저장 & 동기화                            │
│  ⌘S         저장 (Linter 자동 실행)          │
│  ⌘⇧C        Git Commit                      │
│  ⌘⇧P        Git Push                        │
│                                              │
└─────────────────────────────────────────────┘
```

### 워크플로우별 단축키 순서

**아이디어 캡처 (5초)**
```
⌘⇧I → 제목 입력 → Tab → 내용 입력 → Enter
```

**새 포스트 작성 (10초)**
```
⌘N → 제목 입력 → Tab → 슬러그 입력 → Tab → 태그 선택
```

**발행 (5초)**
```
⌘S (저장) → ⌘⇧C (커밋) → ⌘⇧P (푸시)
```

## 8. 모바일 워크플로우

### 모바일 시나리오

```
┌─────────────────────────────────────────────┐
│          📱 모바일 워크플로우                │
├─────────────────────────────────────────────┤
│                                              │
│  1️⃣ 외출 중 아이디어 떠오름                  │
│     ↓                                        │
│  2️⃣ Obsidian 모바일 앱 열기                  │
│     ↓                                        │
│  3️⃣ 메뉴 (⋮) → QuickAdd: 아이디어 캡처      │
│     ↓                                        │
│  4️⃣ 제목 + 내용 간단히 입력                  │
│     ↓                                        │
│  5️⃣ 저장 → 자동 Git 동기화                   │
│     ↓                                        │
│  6️⃣ PC에서 나중에 확장하여 포스트 작성       │
│                                              │
└─────────────────────────────────────────────┘
```

### 동기화 흐름

```
모바일 (외출 중)
    ↓
아이디어 캡처
    ↓
Obsidian Git (자동 푸시)
    ↓
GitHub (클라우드)
    ↓
PC Obsidian (자동 풀)
    ↓
Dashboard에서 확인
    ↓
포스트로 발전
```

```
문제 발생!
    ↓
    ├─ Dataview 작동 안 함

    │       ↓

    │       ↓
    │   해결 ✅
    │

    │       ↓
    │   Obsidian Git 플러그인 활성화 확인
    │       ↓
    │   터미널: git status 확인
    │       ↓
    │   수동 커밋: ⌘⇧C + ⌘⇧P
    │       ↓
    │   해결 ✅
    │
    ├─ 이미지 저장 안 됨
    │       ↓
    │   Paste Image Rename 활성화 확인
    │       ↓
    │   경로 설정: ../public/images/posts
    │       ↓
    │   폴더 존재 확인
    │       ↓
    │   해결 ✅
    │
    └─ 템플릿 작동 안 함

        templates/ 폴더 존재 확인

        해결 ✅
```
